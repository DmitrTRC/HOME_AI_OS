# Словарь терминов (Glossary)

Объяснение современных терминов для человека с опытом из 90-х.
Аналогии с тем что ты уже знаешь.

---

## Инфраструктура

### Docker / Container (Контейнер)
**Что это:** Изолированная среда для запуска приложения со всеми зависимостями.
**Аналогия из 90-х:** Как виртуальная машина (VMware), но в 100 раз легче. Не эмулирует железо — использует ядро хост-системы.
**Зачем:** "У меня работает, а у тебя нет" — больше не проблема. Контейнер одинаков везде.

### Docker Compose
**Что это:** YAML-файл описывающий набор контейнеров и как они связаны.
**Аналогия:** Как batch-файл, но для запуска целой инфраструктуры. `docker compose up` = "подними мне БД, Redis, приложение и бота одной командой".

### Docker Volume
**Что это:** Именованное хранилище данных которое переживает перезапуск контейнера.
**Аналогия:** Как монтирование диска. Контейнер удалился — данные в volume остались.

### S3 / Object Storage
**Что это:** Хранилище файлов по HTTP API. Кладёшь файл по ключу (пути), забираешь по ключу.
**Аналогия:** Как FTP-сервер, но с REST API вместо FTP-команд. Масштабируется бесконечно.
**MinIO** — локальная реализация S3 для разработки. **Selectel S3** — то же самое в облаке.

### Reverse Proxy (Caddy / Nginx)
**Что это:** Сервер перед твоим приложением, который обрабатывает HTTPS, маршрутизирует запросы.
**Аналогия:** Как Apache перед CGI-скриптами в 90-х. Caddy = Apache, но с автоматическим HTTPS.

## Python / Backend

### FastAPI
**Что это:** Веб-фреймворк для Python. Создаёшь HTTP-эндпоинты (routes).
**Аналогия:** Как CGI-скрипты, но в 1000 раз удобнее. Автодокументация, валидация, async.

### async / await
**Что это:** Асинхронное программирование. Один процесс обрабатывает много запросов, не блокируясь на I/O.
**Аналогия:** Как select()/poll() в C для сокетов, но на уровне языка. Вместо потоков — корутины.

### Pydantic
**Что это:** Библиотека валидации данных через типы Python.
**Аналогия:** Как struct в C, но с автоматической проверкой типов при присвоении. Если поле int, а пришла строка — ошибка.

### SQLAlchemy
**Что это:** ORM (Object-Relational Mapper) — работа с БД через объекты Python вместо сырого SQL.
**Аналогия:** Как Delphi TTable/TQuery, но мощнее. Пишешь `session.add(note)` вместо `INSERT INTO...`. SQL всё равно генерируется, но ты его не пишешь руками (хотя можешь).

### Alembic
**Что это:** Система миграций БД. Отслеживает изменения в моделях и генерирует SQL для обновления схемы.
**Аналогия:** Как version control, но для структуры таблиц. Добавил поле в модели → Alembic сгенерировал ALTER TABLE.

### Celery
**Что это:** Распределённая очередь задач. Кладёшь задачу → worker забирает и выполняет в фоне.
**Аналогия:** Как cron + message queue. Пользователь отправил фото → HTTP-запрос мгновенно ответил "ок" → отдельный процесс обрабатывает фото в фоне.

### Redis
**Что это:** Key-value хранилище в оперативной памяти. Очень быстрое.
**Аналогия:** Как memcached, но с персистентностью и структурами данных (списки, множества, хеши). Мы используем для: очереди Celery + кэш.

### uvicorn
**Что это:** ASGI-сервер — запускает FastAPI-приложение.
**Аналогия:** Как Apache для PHP или IIS для ASP. Но для Python async.

## AI / ML

### LLM (Large Language Model)
**Что это:** Большая языковая модель (ChatGPT, Claude, и т.д.). Принимает текст → выдаёт текст.
**Зачем нам:** Классификация входящих данных, извлечение сущностей, суммаризация, черновики отчётов.

### Embeddings (Эмбеддинги)
**Что это:** Превращение текста в вектор чисел (массив float). Похожие тексты → похожие векторы.
**Аналогия:** Как хеш-функция, но сохраняющая смысл. "Кот" и "Котёнок" дадут близкие векторы.
**Зачем:** Семантический поиск. Ищешь "домашнее животное" → находишь записи про "кот", "собака", даже если слова "животное" там нет.

### pgvector
**Что это:** Расширение PostgreSQL для хранения и поиска по векторам (embeddings).
**Зачем:** Вместо отдельного сервиса (Pinecone, Qdrant) — всё в одной БД.

### Ollama
**Что это:** Runtime для запуска LLM-моделей локально на твоём сервере.
**Зачем:** Для чувствительных данных (медицина, документы). Данные не уходят в облако.

### Whisper
**Что это:** Модель от OpenAI для распознавания речи (speech-to-text).
**Зачем:** Голосовые из Telegram → текст. Работает локально, бесплатно, хорошо знает русский.

### OCR (Optical Character Recognition)
**Что это:** Распознавание текста на изображениях. Ты это уже знаешь.
**Tesseract** — движок OCR от Google. **EasyOCR** — нейросетевой OCR, лучше для сложных случаев.

## Паттерны и архитектура

### Modular Monolith (Модульный монолит)
**Что это:** Одно приложение, но внутри — строго изолированные модули.
**Аналогия:** Как DLL-библиотеки в одном exe. Каждый модуль можно вынести в отдельный процесс позже.
**Почему не микросервисы:** Для 1-5 пользователей микросервисы — overhead без пользы. Сложность деплоя × количество сервисов.

### Protocol (Python)
**Что это:** Интерфейс в Python (как interface в Java/C#, abstract class в C++).
**Зачем:** AIProvider Protocol → ClaudeProvider и OllamaProvider реализуют одни и те же методы → код вызывает через интерфейс, не зная конкретного провайдера.

### Dependency Injection (DI)
**Что это:** Вместо `provider = ClaudeProvider()` внутри функции — передаём провайдер снаружи.
**Аналогия:** Как callback в C. Не хардкодим реализацию, а подставляем нужную.
**Зачем:** Легко менять Claude на Ollama. Легко подставить mock в тестах.

### JSONB (PostgreSQL)
**Что это:** Колонка в PostgreSQL которая хранит JSON, но с индексами и поиском.
**Зачем:** Доменные поля разные для каждого типа. Вместо 20 таблиц — одна таблица entities + JSONB `meta` для доменных полей. Vehicle meta: `{"vin": "...", "plate": "..."}`. Note meta: `{"mood": "research"}`.

## Работа с Claude Code

### Claude Code
**Что это:** CLI-инструмент от Anthropic. Запускаешь в терминале в папке проекта — Claude читает код, понимает контекст, пишет и редактирует файлы.
**CLAUDE.md** — файл инструкций. Claude Code читает его первым и понимает: что за проект, какой стек, какие конвенции.

### MCP (Model Context Protocol)
**Что это:** Протокол подключения внешних инструментов к Claude. Через MCP Claude может читать файлы, ходить в Google Drive, смотреть календарь.
**Зачем:** Claude Code использует MCP для доступа к файловой системе.
